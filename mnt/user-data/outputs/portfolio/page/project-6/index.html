기존 서비스의 UX 문제
01
예약 단계가 너무 많음
병원 검색 → 진료과 선택 → 병원 선택 → 시간 선택 → 예약 확인. 단계가 많아 이탈률 높음
02
진료과 선택 어려움
의학 지식이 없는 일반 사용자가 증상에 맞는 진료과를 직접 선택하기 어려운 구조
03
위치 기반 검색 불편
현재 위치 기반 병원 검색이 직관적이지 않고 거리·영업 여부 파악이 한 화면에서 어려움
04
고령 사용자 접근성
복잡한 UI 구조와 작은 텍스트로 인해 병원 앱 주요 사용자인 중장년층 이용에 어려움

AI 프롬프트 예시
ChatGPT서비스 분석
Prompt
"굿닥·똑닥 병원 예약 앱의 UX 구조를 분석해줘. 예약 단계, 진료과 선택 방식, 검색 기능 비교. Jakob Nielsen 원칙 적용."
Result
두 서비스 UX 비교 분석표, 공통 문제점 4가지, Nielsen 원칙별 개선 필요 항목 도출
Claude플로우 설계
Prompt
"병원 예약을 증상 선택 기반으로 단순화하는 UX 플로우를 설계해줘. 3단계 이하로 완료 가능. 모바일 퍼스트, 고령 사용자 고려."
Result
증상→추천→예약 3단계 플로우, 증상 분류 카테고리 8개, 접근성 고려 UI 가이드 제안
ChatGPTUI 설계
Prompt
"증상 선택 화면 UI 컴포넌트를 설계해줘. 이모지+텍스트 조합의 증상 카드 그리드. 탭 시 즉시 추천 병원 노출."
Result
증상 카드 컴포넌트, 선택 상태 스타일, 병원 추천 카드 레이아웃, 인터랙션 구조 완성
